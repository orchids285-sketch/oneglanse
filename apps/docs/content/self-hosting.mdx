# Self Hosting

Self-hosting is the third setup path after the hosted cloud app and local mode.
Choose it when you want the full product stack in your own infrastructure.

OneGlanse now runs as two Docker stacks:

- the app stack in `docker-compose.yml`
- the public stack in `docker-compose.public.yml`

That separation keeps landing and docs online while you redeploy the main app.

## Start the Stack

```bash
pnpm install
pnpm self-host
```

That creates env files if missing, ensures the shared edge network exists, and
starts both stacks.

## App Stack

- `web`
- `agent-worker`
- `redis`
- `db`
- `clickhouse`
- `migrate`

This is the default stack, so raw `docker compose up -d --build` and
`docker compose down` only affect these services.

## Public Stack

- `landing`
- `docs`
- `app-gateway`

The app gateway stays online and serves a short maintenance page on port `3001`
when the `web` container is unavailable.

## Default Ports

- `3000` - landing
- `3001` - app gateway
- `3002` - docs
- `3333` - auth upload API

## Common Deployment Commands

Initial VPS setup:

```bash
pnpm self-host
```

Routine app-only update:

```bash
docker compose up -d --build
```

Or:

```bash
pnpm self-host:app
```

Public-only update:

```bash
docker compose -f docker-compose.public.yml up -d --build
```

Or:

```bash
pnpm self-host:public
```

## Persistent Storage

The self-hosted stack persists auth and runtime state on the host at:

```text
/opt/oneglanse/storage
```

Containers see the same storage at:

```text
/storage
```

This includes:

- auth bundles
- auth status files
- persistent runtime browser profiles

## What You Usually Need to Configure

In most deployments, only a few variables matter:

- `OPENAI_API_KEY`
- `BETTER_AUTH_SECRET`
- `APP_URL`
- `API_BASE_URL`
- `AGENT_AUTH_UPLOAD_TOKEN`

Optional:

- Google OAuth values
- proxy settings

## Auth in Self-Hosted Mode

Interactive provider login never runs on the VPS.

Use the local auth flow:

```bash
pnpm auth -- --upload-url http://YOUR_VPS_HOST:3333/auth/sessions --upload-token YOUR_TOKEN
```

That pushes the auth bundle to the VPS and lets the runtime reseed from it on
the next run.
