# Environment Variables

Most variables already have defaults. The repo is designed so local setup and
self-hosting work with very little manual configuration.

## `.env`

These are the variables most users will care about.

### Core

- `DATABASE_URL`
- `CLICKHOUSE_URL`
- `REDIS_HOST`
- `REDIS_PORT`
- `REDIS_PASSWORD`

These already have good local defaults in `.env.example`.

### App

- `BETTER_AUTH_SECRET`
- `APP_URL`
- `API_BASE_URL`
- `ONEGLANSE_APP_MODE`
- `OPENAI_API_KEY`

### Optional login integrations

- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

### VPS auth sync

- `AGENT_AUTH_UPLOAD_TOKEN`

This is the token the VPS auth upload API expects.

### Optional VPS proxying

- `PROXY_PROVIDER`
- `PROXY_SCHEME`
- `PROXY_HOST`
- `PROXY_PORT`
- `PROXY_USERNAME`
- `PROXY_PASSWORD`
- `THORDATA_PROXY_API_URL`

Proxy settings are only used on the VPS runtime path.

## `apps/agent/.env`

This file is intentionally small.

Most users only need it for browser/runtime overrides:

- `CAMOUFOX_HEADLESS_MODE`
- `CAMOUFOX_PYTHON_BIN`
- `DEBUG_ENABLED`

## Local Auth Upload

For local-to-VPS auth capture, prefer passing the upload destination directly
to `pnpm auth`:

```bash
pnpm auth -- --upload-url http://YOUR_VPS_HOST:3333/auth/sessions --upload-token YOUR_TOKEN
```

That is cleaner than editing env files just for a one-off auth sync.

## Variables You Usually Do Not Need to Touch

These are already handled by defaults or Docker wiring:

- `AGENT_AUTH_ROOT_DIR`
- `AGENT_API_HOST`
- `AGENT_API_PORT`
- Postgres/ClickHouse internal container paths
- the VPS storage bind mount path inside containers (`/storage`)

## Recommended Minimums

### Local

Edit only when needed:

- `OPENAI_API_KEY`
- `BETTER_AUTH_SECRET`

### VPS

Set these:

- `OPENAI_API_KEY`
- `BETTER_AUTH_SECRET`
- `APP_URL`
- `API_BASE_URL`
- `AGENT_AUTH_UPLOAD_TOKEN`

Optional:

- Google OAuth values
- proxy values
