# Getting Started

## Requirements

- Node.js 20+
- pnpm 10+
- Docker + Docker Compose

## Install

```bash
pnpm install
```

## Start the Full Local App

```bash
pnpm local
```

`pnpm local` is the main local entrypoint. It:

- creates `.env` and `apps/agent/.env` if they do not exist
- bootstraps a pinned local Camoufox package/browser pair automatically on first run
- starts PostgreSQL, ClickHouse, and Redis in Docker
- runs the database migrations
- starts the web app and the agent locally
- opens the app at `http://localhost:3000`

## Connect Providers

If auth is missing, the app redirects to `/providers`.

You can also start only the auth flow:

```bash
pnpm auth
```

That opens the shared providers UI at
`http://localhost:3000/providers`.

Connect flow:

1. Click a provider button
2. A local Camoufox sign-in browser opens
3. Sign in
4. Close the browser when finished
5. The UI shows a checkmark when the auth session is saved

## Self-Hosted Deploys

For VPS use, `pnpm self-host` starts both:

- the app stack
- the public stack

Routine product changes should then use the default app stack only:

```bash
docker compose up -d --build
```

That keeps landing and docs online during app redeploys.

## Supported Providers

Runtime providers:

- ChatGPT
- Perplexity
- Gemini
- Claude
- AI Overview

Auth groups:

- ChatGPT
- Perplexity
- Google
- Claude

Gemini and Google Search auth are stored separately.

## Local Defaults

Local mode is designed to work with minimal configuration.

- proxies are not used
- auth bundles are stored under `.oneglanse-storage/auth`
- persistent runtime profiles are stored under `.oneglanse-storage/runtime`
- runtime profiles are bootstrapped from saved auth bundles when needed

## Most Common Local Variables

Most users only need these in `.env`:

- `OPENAI_API_KEY`
- `BETTER_AUTH_SECRET`
- `APP_URL`
- `API_BASE_URL`
- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

And in `apps/agent/.env`, only the values your local worker actually needs,
typically Redis and any proxy credentials.
