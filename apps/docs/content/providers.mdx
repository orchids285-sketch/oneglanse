# Providers

## Runtime Providers

OneGlanse currently runs prompts against:

- ChatGPT
- Perplexity
- Gemini
- Claude
- AI Overview

These are the providers the worker executes at runtime.

## Auth Groups

The auth model is intentionally grouped:

- ChatGPT
- Perplexity
- Gemini
- Google
- Claude

Gemini and Google Search now use separate auth bundles so their login flows stay
independent.

Google auth is used by:

- AI Overview

Gemini auth is used by:

- Gemini

## Connect Flow

The provider-connections module is shared between:

- `/connections`
- `/provider-connections`

Each connect action:

1. launches a local Camoufox sign-in browser
2. waits for an authenticated session
3. saves a compact auth bundle for that auth group
4. shows a completed checkmark in the UI

## Runtime Reuse

At runtime, the worker:

- loads the auth bundle for the auth group
- seeds the runtime profile if needed
- otherwise reuses the provider’s persistent runtime profile directly

That keeps local and VPS execution consistent without replaying auth state on
every launch.
