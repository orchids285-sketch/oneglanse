# Proxy Setup

Proxy settings apply to `agent-worker`.

From `apps/agent/src/env.ts`:

- `PROXY_SOURCE_MODE` (default `auto`)
- `PROXY_API_URL` (optional)
- `PROXY_MANUAL_FILE` (optional)
- `PROXY_CACHE_TTL_MS` (default `10000`)

## Mode 1: API-backed Proxy Source

Use when your proxy vendor exposes an HTTP API endpoint.

```env
PROXY_SOURCE_MODE=api
PROXY_API_URL=https://proxy-provider.example.com/list
```

## Mode 2: Manual File Source

Use when you manage your own proxy list file.

```env
PROXY_SOURCE_MODE=manual
PROXY_MANUAL_FILE=/storage/proxies.txt
```

`/storage` is persisted via the `agent_storage` Docker volume.

## Mode 3: Auto

```env
PROXY_SOURCE_MODE=auto
```

Auto mode uses whichever source variables are present.

## Operational Guidance

- Keep proxy credentials out of source control.
- Track provider-specific failure rates.
- Increase `AGENT_WORKER_CONCURRENCY` only after proxy pool stability is verified.
