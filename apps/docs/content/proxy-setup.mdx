# Proxy Setup

Proxy settings are only relevant for the VPS runtime path.

Local mode runs direct by default.

## Supported Inputs

Split proxy fields:

```env
PROXY_SCHEME=http
PROXY_HOST=proxy.example.com
PROXY_PORT=8080
PROXY_USERNAME=user
PROXY_PASSWORD=pass
```

## ThorData

ThorData can use the API discovery path:

```env
PROXY_PROVIDER=thordata
PROXY_SCHEME=http
THORDATA_PROXY_API_URL=https://get-ip.thordata.net/api?td-customer=YOUR_CUSTOMER_ID&sesstype=1&number=5&country=US
```

Runtime behavior:

- the worker fetches the proxy list at launch time
- picks one returned proxy
- avoids leasing the same returned proxy to multiple workers concurrently

## Important Rules

- `PROXY_HOST` and `PROXY_PORT` must be set together
- `PROXY_USERNAME` and `PROXY_PASSWORD` must be set together
- `THORDATA_PROXY_API_URL` is only valid with `PROXY_PROVIDER=thordata`

## Recommended Usage

Only enable proxies on the VPS when needed.

Most self-hosted users should start without proxying, validate that the stack is
working, then add proxy settings if the provider runtime needs them.
