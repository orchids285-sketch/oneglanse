## Local

Use local mode to run the full stack on your own machine — interactive provider auth, manual prompt runs, all data stored locally.

**Requirements:** Node.js 20+, pnpm 10+, Docker

```bash
git clone https://github.com/aryamantodkar/oneglanse
cd oneglanse
pnpm install
cp .env.example .env
```

Open `.env` and set your LLM API key — this is the only value you must provide:

```bash
# Pick one:
OPENAI_API_KEY=sk-...

# or to use Claude:
ANTHROPIC_API_KEY=sk-ant-...
ANALYSIS_LLM_PROVIDER=claude
```

Then start:

```bash
pnpm local
```

Opens at `http://localhost:3000`. On first run: starts Postgres, ClickHouse, and Redis, runs migrations, and bootstraps the browser runtime. Sign up with email — Google OAuth is optional.

Go to `/providers` to connect your AI accounts, then add prompts and run.

## Self-Host

Self-host mode runs the full stack on a VPS with recurring scheduled runs. Provider auth is captured locally and transferred to the VPS — the server itself never opens a browser window.

### Step 1 — Deploy to VPS

On your VPS, clone the repo and start the stack:

```bash
git clone https://github.com/aryamantodkar/oneglanse
cd oneglanse
pnpm install
cp .env.example .env
```

Open `.env` and fill in:

```bash
# Required: LLM key for response analysis
OPENAI_API_KEY=sk-...          # or ANTHROPIC_API_KEY + ANALYSIS_LLM_PROVIDER=claude

# Required: residential proxy (see below)
THORDATA_PROXY_API_URL=https://your-thordata-endpoint

# Required: token used to authenticate auth session uploads from your local machine
AGENT_AUTH_UPLOAD_TOKEN=your-secret-token

# Optional: public URL of your VPS app
APP_URL=https://your-domain.com
API_BASE_URL=https://your-domain.com
```

Then start:

```bash
pnpm self-host
```

This starts the app stack and the public Nginx gateway. Default ports:

- App: `3001`
- Auth upload API: `3333`

Persistent state is stored at `/opt/oneglanse/storage`. Override with `ONEGLANSE_STORAGE_ROOT`.

### Step 2 — Set up a residential proxy

AI providers block datacenter IP addresses. On VPS you must route browser traffic through a residential proxy so requests appear to come from a real user location.

OneGlanse uses **[ThorData](https://thordata.com)** residential proxies. Sign up for a ThorData account and get your rotating proxy API endpoint URL.

Add it to your VPS `.env`:

```bash
THORDATA_PROXY_API_URL=https://your-thordata-api-endpoint
```

Before each browser launch, OneGlanse calls this URL, receives a list of available residential proxies, picks one, and routes all browser traffic through it. The proxy is released after the run completes.

### Step 3 — Connect provider accounts

Provider auth requires an interactive browser window — it cannot run on a headless server. You capture sessions on your **local machine** and upload them to the VPS.

**On your local machine**, add these to your local `.env`:

```bash
ONEGLANSE_VPS_IP=your-vps-ip
AGENT_AUTH_UPLOAD_TOKEN=your-secret-token   # must match the VPS value
```

If you already have local sessions saved, upload them:

```bash
pnpm upload:vps
```

If you need to capture sessions first:

```bash
pnpm auth
```

This opens the local `/providers` page. Connect each provider interactively, then run `pnpm upload:vps` to transfer the sessions to your VPS.

Re-run this whenever a provider session expires.

### Step 4 — Verify

Check that the agent on the VPS is receiving the sessions:

```bash
curl http://YOUR_VPS_IP:3333/health
```

Then go to your app URL, add prompts, and start a scheduled run.

### Updates

```bash
pnpm self-host:app      # Pull and restart app stack only
pnpm self-host:public   # Pull and restart public gateway only
```
